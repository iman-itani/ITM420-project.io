Use-Case Name:
Generate general report
Use Case Type
Business Requirements ��
Use-Case ID:
Priority:
Source:
Interview 1
Primary System Actor:
Financial auditor
Description:
Generate various types of reports, such as reports by service, reports by doctor, and reports related to donor contributions.
Precondition:
The financial auditor must have a valid login credential to access the system and must be authorized to generate reports.
Trigger:
 Identification of a need for summarized reports.
This trigger can come from various sources, such as doctor’s requests, patients’ requests, general management inspections, or internal business strategies.
Typical Course of Events:
Actor
System response

 
Step 1:
The financial auditor accesses the system's management interface and navigates to the section of report generation.
Step 3: The auditor sets the filter with the required or intended information and submits it.
 Step 2: The system displays a list of filters to select and checkboxes to select to proceed.
Step 4: The system processes the request and verifies that all selected information is valid
Step 5: the system retrieves the data using the Extract Data use case by considering settled filters as primary keys.
Step 6: Once the request is submitted and validated, the system generates the report upon the information (checkboxes and filters) selected.
Alternate Courses:
Alternate to step 4:  If the information is incomplete or invalid, the financial auditor must check with the database user and adjust the data accordingly.
Conclusion:
Upon successful completion of the use-case, the requested report is generated and displayed to the financial auditor.
Postcondition:
The requested report is generated and displayed to the financial auditor.
Business Rules:
Only authorized personnel should be able to generate reports.
Implementation Constraints and Specifications:
The system must have a backup and recovery mechanism in case of data loss or system failure.
Assumptions:
 The financial auditor enters the correct parameters for the report.
The system can handle multiple users generating reports simultaneously
