Use-Case Name:
Issue payroll for doctors
 Use Case Type
Business Requirements þ
Use-Case ID:
Priority:
Source:
Interview 2
Primary System Actor:
Accountant
Other Interested Stakeholders:
Doctor
Description:
This use case involves calculating and issuing payroll for doctors based on the number of services they provided to the patients.
Precondition:
The doctor's services have been recorded in the system.
Trigger:
The 20th of each month.
Typical Course of Events:
System response

 
 Step 1: The system extracts data related to the services provided by each doctor during the current month, using extract data use case.
Step 2: The system calculates the salary of each doctor based on the number of services they provided.
Alternate Courses:
 Alt step 1: If there is a discrepancy in the data extracted by the system or the system is unable to extract the data
  • The accountant investigates and corrects the data manually and when needed retrieves the required data and initiates the process manually.
Conclusion:
Payroll for doctors has been calculated and issued on time.
Postcondition:
Payroll records are updated in the system.
Business Rules:
 The payroll is issued on the 20th of each month.
The salary of each doctor is calculated based on the number of services they provided.
Assumptions:
 The system accurately records the services provided by each doctor.
The accountant approves the generated payroll and issues it to each doctor.
