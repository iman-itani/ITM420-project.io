Use-Case Name:
Log in
 Use Case Type
Business Requirements þ
Use-Case ID:
Priority:
Source:
Interview 1
Primary System Actor:
User
Description:
This use-case allows the user to log in, access and navigate the system.
Precondition:
The user must enter valid login credentials to gain access to the system.
Trigger:
 The cashier may want to extract, enter, or/and modify data.
The administrator may want to modify records or/and customize the system.
The accountant may want to issue invoice, receipt or/and voucher
Typical Course Of Events:
Actor
System response

 
 Step 1: The user navigates to the system login page and enters his unique username and password.


 
 Step 2: The system verifies the user's login credentials
Step 3: The system grants access to the user if the login credentials are valid.
Step 4: The user is redirected to the system homepage or the last page they were on before logging out.
Alternate Courses:
 Alternate to step 3:  If the user enters an invalid username or password, the system displays an error message and prompts the user to try again.
If the user forgets their password, they can request a password reset by following the appropriate steps.
Conclusion:
The user is able to successfully log in and access the subsystem.
Postcondition:
The user is logged in and able to access the system's features and functionalities.
Business Rules:
 The user must have valid login credentials to access the system.
The system must be secure and protect user information.
Implementation Constraints and Specifications:
 The login page must be accessible and user-friendly.
The system must have measures in place to prevent unauthorized access.
Assumptions:
 The user has a valid username and password.
The user is authorized to access the system.
