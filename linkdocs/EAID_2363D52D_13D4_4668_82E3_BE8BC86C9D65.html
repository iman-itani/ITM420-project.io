Use-Case Name:
Generate financial reports
Use Case Type
Business Requirements þ
Use-Case ID:
Priority:
Source:
 Interview 2
 
Primary System Actor:
Financial auditor (primary system)
Description:
Generating financial reports that provide data on the financial aspects of the clinic such as net income statement, and balance sheet.
Precondition:
The financial auditor must have a valid login credential to access the system and must be authorized to generate financial reports.
Trigger:
A report should be generated to keep track of the financial standing of the medical center.
Typical Course of Events:
Actor
System response

 
 Step 1: Accesses the system's management interface and navigates to the section of report generation
Step 3: The accountant initiates the generation of the report.
Step 6: The accountant confirms the request.
 Step 2: The system displays the list of financial reports that could be generated
Step 4: the system generates the financial report.
Step 5: The system processes the request and verifies that all required information selected by the financial auditor is valid.
Step 7: The system extract data needed for the financial report.
Step 8: The system generates the report.
Alternate Courses:
 Alternate to step 6:  The request is cancelled
  • The system go back to the home page.
Conclusion:
This use case concludes when a financial report is generated.
Postcondition:
The financial auditor gains insights into the financial status of the system.
Business Rules:
Only authorized personnel should be able to generate financial reports.
Implementation Constraints and Specifications:
The system must have a backup and recovery mechanism in case of data loss or system failure.
Assumptions:
 The financial auditor is able to interpret and analyze the financial report.
The financial report provides accurate and up-to-date financial information.
