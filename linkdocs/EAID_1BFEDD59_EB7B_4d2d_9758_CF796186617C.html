Use-Case Name:
Issue payroll
Use Case Type
Business Requirements þ
Use-Case ID:
Priority:
Source:
Interview 2
Primary System Actor:
Accountant
Other Interested Stakeholders:
Doctor, contractual employee
Description:
This use case involves calculating and issuing payroll for doctors based on the number of services they provided to the patients.
Precondition:
The doctor's services have been recorded in the system.
Trigger:
The 20th of each month.
Typical Course of Events:
Actor
System response

 
 Step 1: The accountant logs onto the system to issue the payroll.
Step 4: The accountant verifies the accuracy of the report, approves the report and initiates the payment process.
Step 6: The accountant issues the payroll to each doctor.
 Step 2: if the payroll is issued for a doctor, use payroll for doctors use case.
Step 3: The system generates a report of the calculated salary for each doctor.
Step 5: The system generates a payroll for each doctor.
Alternate Courses:
 Alt step 2: If not, use payroll for contractual employee.
Alt step 4: If there is an error in the calculation of the salary or the report is inaccurate, the accountant investigates and corrects the error.
Alt step 6: If the payroll is not received by a doctor, the accountant investigates and resolves the issue.
Conclusion:
Payroll for doctors has been calculated and issued on time.
Postcondition:
Payroll records are updated in the system.
Business Rules:
 The payroll is issued on the 20th of each month.
The salary of each doctor is calculated based on the number of services they provided.
Assumptions:
 The system accurately records the services provided by each doctor.
The accountant approves the generated payroll and issues it to each doctor.
